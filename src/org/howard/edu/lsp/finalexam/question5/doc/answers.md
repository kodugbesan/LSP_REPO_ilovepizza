Heuristic 1:
Classes should be small and focused on a single responsibility.

Explanation:
This improves readability because each class has a clear purpose and is easier to understand. It also improves maintainability since changes to one part of the system are less likely to affect other parts.

Heuristic 2:
Hide internal data and implementation details.

Explanation:
Encapsulation makes code easier to maintain because other parts of the program do not depend on internal details. It also reduces the risk of errors since data is controlled through methods instead of being accessed directly.

Heuristic 3:
Avoid large methods that try to do too much.

Explanation:
Breaking methods into smaller pieces makes the code easier to read and debug. It also makes it easier to reuse parts of the code and modify behavior without affecting the entire method.
